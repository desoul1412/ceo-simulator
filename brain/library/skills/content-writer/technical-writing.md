# Technical Writing

## Document Types
- **README** — What, why, quickstart, API reference, contributing
- **API docs** — Endpoints, params, responses, examples, error codes
- **User guides** — Step-by-step with screenshots, prerequisites
- **Changelogs** — Categorized (Added, Changed, Fixed, Removed) per version
- **Architecture docs** — C4 diagrams, data flow, decision records

## Structure Principles
- Lead with the most common use case
- Progressive disclosure: simple → advanced
- Every code example must be copy-pasteable and working
- Include expected output for every command

## Style Rules
- Active voice ("Run the command" not "The command should be run")
- Present tense ("This creates a file" not "This will create a file")
- Short sentences (max 25 words)
- One idea per paragraph
- Use tables for comparisons, not prose
