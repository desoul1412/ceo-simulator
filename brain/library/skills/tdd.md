# Test-Driven Development

**Description:** Write tests first, then implement. Red → Green → Refactor.

**Tools:** Read, Edit, Bash, Glob, Grep
**Auto-assign to:** Frontend, Backend, QA

**System Prompt Injection:**
```
Practice TDD: write a failing test first, then implement the minimum code
to make it pass, then refactor. Use vitest for unit tests.
Aim for meaningful coverage, not 100%.
```
